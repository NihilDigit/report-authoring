# 实验报告（中国高校计算机类）

## 典型结构（模板一般由老师给，这里是常见通用框架）

```
【顶部元信息】
- 课程号、课程名
- 班级 / 学号 / 姓名 / 教师
- 实验编号 / 实验名称

【主体表格】
┌─────────────┬─────────────────────────────┐
│ 实验名称      │ 实验 X XXX                   │
├─────────────┼─────────────────────────────┤
│ 实验目的与要求 │ （老师已填，几条要求）        │
├─────────────┼─────────────────────────────┤
│ 实验过程      │ ★ 学生填写（主战场）           │
├─────────────┼─────────────────────────────┤
│ 结果分析      │ ★ 学生填写                     │
└─────────────┴─────────────────────────────┘
```

## 实验过程 — 常见写法

按「实验目的与要求」里的点**逐条交代**：

```markdown
一、开发环境与项目结构
  - 操作系统、工具链版本、依赖安装
  - 项目目录 tree 图

二、要求 1 的实现
  1. 自定义数据结构 / 算法设计
  2. 源代码关键片段
  3. 编译 / 运行 / 验证
  4. 截图

三、要求 2 的实现
  （同上）

……
```

每条要求**三件套**：
1. 设计思路（文字）
2. 代码片段 / 命令（silicon 图或 code block）
3. 运行截图 + caption

## 结果分析 — 常见写法

多段文字，覆盖：
1. **整体完成情况**：哪些完成、哪些选做也做了
2. **对比 / 观察**：用表格列 key 数据
3. **讨论**：优缺点、权衡、业界实际场景
4. **遇到的问题与解决**：每条「触发 - 表现 - 解决 - 原因」
5. **实验体会**：个人对知识点的融会贯通

语气：第一人称过去式为主（「本次实验完成了…」、「我发现…」）；避免口语化。

## 默认排版约定

见 `docs/typography.md`。关键：
- 正文：宋体五号 1.5 倍行距首行缩进两字
- 小标题：加粗小四
- 大标题：加粗小三
- 代码：Consolas 小五
- Caption：加粗小五居中

## 图表数量参考

- 截图：每个要求 1~3 张关键运行图
- 代码图：自定义结构 / 关键算法 / 关键调用，每章 3~8 张
- 表格：对比性结果必出表（不同格式字节数、不同算法耗时等）

## 篇幅

一次实验的报告通常 10~25 页（含所有截图）。太短怀疑做得不够细，太长可能没抓重点。
